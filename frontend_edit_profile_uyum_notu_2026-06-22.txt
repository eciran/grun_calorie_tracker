GRUN FRONTEND - EDIT PROFILE UYUM NOTU
Tarih: 2026-06-22
Konu: Edit Profile ekraninda guncellenebilen alanlar ve backend profil API kapasitesi

1) MEVCUT FRONTEND DURUMU

Frontend EditProfile ekrani su anda kullaniciya temel olarak asagidaki alanlari duzenletiyor:

- Full name
- Age
- Current weight
- Height
- Avatar upload/delete

Ilgili ekran:
C:\Users\emrah\IdeaProjects\Claude_Grun_frontend\src\screens\EditProfile.tsx

Frontend'in PUT /api/v1/users/me icin gonderdigi payload pratikte su alanlarla sinirli:

- name
- age
- weight
- height

Avatar ayri endpointlerle yonetiliyor:

POST   /api/v1/users/me/avatar
DELETE /api/v1/users/me/avatar

2) BACKEND'IN KABUL ETTIGI PROFIL ALANLARI

Backend UserProfileDto ve UserServiceImpl.updateCurrentUser() incelendiginde PUT /api/v1/users/me endpointi su alanlari guncelleyebiliyor:

- name
- gender
- age
- height
- weight
- bodyFat
- bmi
- marketRegion
- preferredLanguage
- timeZone
- unitPreference

Backend response icinde ayrica su alanlar da donuyor:

- id
- email
- role
- emailVerified
- passwordSet
- accountEnabled
- accountLocked
- avatarUrl
- goalRecalculationRecommended
- goalRecalculationReason

3) EDIT PROFILE ICIN EKLENMESI ONERILEN ALANLAR

Edit Profile ekrani "kisisel fiziksel profil" olarak ele alinmali. Bu nedenle asagidaki alanlar Edit Profile icinde yer alabilir:

- Full name
- Gender
- Age veya birthday
- Height
- Current weight
- Body fat, opsiyonel/gelişmis alan
- Avatar

En kritik eksik: gender.

Neden onemli?
Backend kalori, BMI/body fat ve hedef hesaplarinda gender bilgisini kullaniyor. Kullanici onboarding sonrasi gender bilgisini duzeltemezse hesaplamalar yanlis kalabilir.

4) EDIT PROFILE ICINDE OLMAMASI DAHA DOGRU OLAN ALANLAR

Asagidaki alanlar backend profile DTO icinde yer alsa da urun rolu olarak Edit Profile icinde olmamali; Settings altinda ayri preference ekranlarinda kalmali:

- marketRegion
- preferredLanguage
- unitPreference
- timeZone

Onerilen konumlar:

- marketRegion -> Language & Region ekraninda
- preferredLanguage -> Language & Region ekraninda
- unitPreference -> Settings / Preferences icinde, kalici backend baglantisiyla
- timeZone -> otomatik cihaz timezone veya Language & Region altinda ileri ayar

5) BODY FAT / BODY METRICS AKISI

Backend'de su endpoint mevcut:

POST /api/v1/users/calculate-body-fat-or-bmi

Frontend authService icinde calculateBodyMetrics metodu var. Backend su olculeri kabul ediyor:

- waistCircumference
- neckCircumference
- hipCircumference, kadin kullanici icin gerekli olabilir

Bu alanlari Edit Profile ana formuna direkt koymak zorunlu degil. Daha iyi UX:

- Edit Profile icinde "Body composition" bolumu olabilir.
- Body fat alani opsiyonel gosterilebilir.
- Daha detayli olcumler icin ayri modal/sheet acilabilir.
- Kullanici olcum girmezse backend BMI tabanli tahmine dusebilir.

MVP icin body fat manuel/opsiyonel kalabilir. Gender ise temel alan olarak eklenmeli.

6) GOAL RECALCULATION DAVRANISI

Backend updateCurrentUser() su alanlarda degisiklik olursa response icinde goalRecalculationRecommended=true donebilir:

- gender
- age
- height
- weight
- bodyFat

Backend response:

- goalRecalculationRecommended
- goalRecalculationReason

Frontend su anda EditProfile save sonrasi sadece profile'i store'a yazip geri donuyor gibi gorunuyor. Bu yeterli degil.

Frontend'de yapilmasi gereken:

- PUT /api/v1/users/me response'u kontrol edilmeli.
- response.goalRecalculationRecommended === true ise kullaniciya net CTA gosterilmeli.
- CTA ornegi: "Your body metrics changed. Recalculate your calorie goal."
- CTA hedefi: /edit-goals veya /setup/review benzeri hedef yeniden hesaplama ekrani.

Bu davranis onemli cunku kilo, boy, yas veya gender degisince mevcut kalori/makro hedefleri eski kalabilir.

7) EMAIL VE PASSWORD KONUSU

Email Edit Profile icinde duzenlenmemeli.

Neden?
Backend'de email change endpoint'i yok. AccountSecurity tarafinda email read-only tutulmasi dogru karar.

Password Edit Profile icinde olmamali.

Dogru konum:
AccountSecurity -> Change Password

8) FRONTEND ICIN ONCELIKLI AKSIYONLAR

Yuksek oncelik:

1. EditProfile ekranina gender eklenmeli.
2. EditProfile save response'unda goalRecalculationRecommended kontrol edilmeli.
3. Recalculation gerekiyorsa kullanici /edit-goals veya hedef review akisina yonlendirilmeli.
4. Settings icindeki unitPreference local state olmaktan cikarilip backend profile update veya ayri preference servisiyle kalici hale getirilmeli.

Orta oncelik:

5. Body fat opsiyonel olarak EditProfile icine eklenebilir.
6. Body composition icin waist/neck/hip modal/sheet tasarlanabilir.
7. timeZone otomatik cihaz bilgisinden backend'e gonderilebilir.

Dusuk oncelik:

8. marketRegion ve preferredLanguage EditProfile'a eklenmemeli; mevcut LanguageRegion ekraninda kalmali.

9) NET URUN KARARI


Settings/Preferences = uygulama tercihleri.
EditProfile = kullanicinin kisisel/fiziksel profili.
AccountSecurity = hesap, sifre, veri ve gizlilik.
EditGoals = hedef, aktivite seviyesi ve haftalik kilo degisim hizi.

Bu ayrim korunursa Profile/Settings yapisi daha anlasilir olur ve backend ile daha tutarli calisir.
